# GPT-5.4 Backend Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Switch the assistant backend default model to `gpt-5.4` and support GPT-5.4 reasoning values without breaking legacy reasoning env values.

**Architecture:** Update the Node and PHP backend config parsing in place, keep the Responses API payload format unchanged, and lock the behavior down with focused tests plus refreshed docs and example environment config. The implementation stays narrow: no frontend behavior changes, no request-shape changes, and no unrelated refactors.

**Tech Stack:** Node.js, native `node:test`, Express, PHP, Markdown docs, dotenv config

---

**Step 1: Write the failing test**

```javascript
import assert from 'node:assert/strict'

describe('server-reasoning-config', () => {
    it('accepts gpt-5.4 reasoning values', () => {
        assert.equal(ServerReasoningConfig.parseReasoningEffort('none', 'minimal'), 'none')
        assert.equal(ServerReasoningConfig.parseReasoningEffort('xhigh', 'minimal'), 'xhigh')
    })
})
```

**Step 2: Run test to verify it fails**

Run: `npm test -- tests/server-reasoning-config.test.mjs`
Expected: FAIL because `ServerReasoningConfig` does not exist yet.

**Step 3: Write minimal implementation**

```javascript
export class ServerReasoningConfig {
    static parseReasoningEffort(rawValue, fallback) {
        const normalized = String(rawValue || '').trim().toLowerCase()
        if (['none', 'minimal', 'low', 'medium', 'high', 'xhigh'].includes(normalized)) {
            return normalized
        }
        return fallback
    }
}
```

**Step 4: Run test to verify it passes**

Run: `npm test -- tests/server-reasoning-config.test.mjs`
Expected: PASS

**Step 5: Commit**

```bash
git add tests/server-reasoning-config.test.mjs src/ServerReasoningConfig.mjs src/server.mjs
git commit -m "test: add backend reasoning config parser"
```
